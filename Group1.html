About Java

Java is a high level, modern programming language designed in the early 1990s by Sun Microsystems, and is now currently owned by Oracle.

Java is platform-independent, which means that you only need to write the program once to be able to run it on a number of different platforms.

Java is portable, with the capability to fit the needs of virtually any application.

Java is used to develop applications for Google's Android OS, various desktop applications, (such as media players and antivirus programs), web applications (such as interactive content), enterprise applications (i.e., banking), and many more.

Some key functions include:

Object Oriented -
Everything in Java is an object which is very modular and reusable based on other factors like for instance encapsulation, inheritance or polymorphism.

Simple -
Java is very easy to learn, and its syntax is simple, for ease of understanding makes Java codes available for learning or using for developers – beginners or advanced developers.

Robust -
Java has a clear focus on the control of memory space – automatic garbage collector as well as exceptions that minimize the possibility of complete system failures.

Multithreading -
this feature enabled the running of many threads at the same time in a given program making good use of the available resources.

High Performance -
By JIT compilation, the intermediate code – Java bytecode is only converted at runtime to native machine code for enhanced execution speed.

Dynamic and Distributed -
Dynamic and Distributed: There is constant linking and loading of classes in Java during execution of Java applications.

Secure -
Java is very secure as it has some secured bytecode that Refers, Has, Exhibits & Possesses absence of direct pointers etc.

Some security measures are as follows:

  • Java programs are run inside a virtual machine known as a sandbox.
  • Java uses a Byte-code verifier that checks for illegal code that can violate access right to object. It provides library level safety.
  • Run-time security check only takes place when we load a new code.

    • These is only the tip of the iceberg on what makes Java so Secure.
